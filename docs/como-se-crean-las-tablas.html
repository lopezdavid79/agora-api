Como se crean las tablas en Agora API

Programa Agora Argentina — Backend REST · Express + Sequelize + MySQL

Este documento explica el flujo completo de creacion de tablas en el servidor. El proyecto usa Sequelize ORM con MySQL y un sistema de migraciones versionadas con Umzug.

Flujo general al iniciar el servidor

sequelize.sync() → umzug.pending() → umzug.up() → Servidor listo

El codigo que orquesta esto esta en src/server.js, funcion iniciar():

async function iniciar() {
  await sequelize.authenticate();
  // -- 1. Crea tablas si no existen --
  await sequelize.sync({ force: false });

  // -- 2. Migraciones versionadas --
  const pendientes = await umzug.pending();
  if (pendientes.length > 0) {
    await umzug.up();
  }

  // -- 3. Arranca el servidor --
  app.listen(PORT, ...);
}

Etapa 1 — sequelize.sync()

sequelize.sync({ force: false }) crea las tablas que no existen basandose en las definiciones de los modelos. No borra ni modifica tablas existentes.

Ejemplo de definicion de modelo

src/models/Usuario.js 
const Usuario = sequelize.define('Usuario', {
  id:       { type: DataTypes.INTEGER, primaryKey: true, autoIncrement: true },
  dni:      { type: DataTypes.STRING(20), allowNull: false, unique: true },
  email:    { type: DataTypes.STRING(100), allowNull: false, unique: true },
  password: { type: DataTypes.STRING(255), allowNull: false },
  rol:      { type: DataTypes.ENUM('Candidato','Gestor','Coordinador','GestorTecnico','Administrador'),
              defaultValue: 'Candidato' },
  activo:   { type: DataTypes.BOOLEAN, defaultValue: true },
}, {
  tableName: 'usuarios',
  timestamps: true,
  createdAt: 'fecha_registro',
  updatedAt: false,
});

Etapa 2 — Migraciones con Umzug

Despues del sync(), se ejecutan las migraciones versionadas con Umzug. Esto permite cambios que sync() solo no puede manejar: agregar columnas a tablas existentes, insertar datos iniciales (seeds), etc.

Configuracion de Umzug

src/config/umzug.js 
const umzug = new Umzug({
  migrations: {
    glob: path.join(__dirname, '..', 'migrations', '*.js'),
  },
  context: sequelize.getQueryInterface(),
  storage: new SequelizeStorage({ sequelize }),
  logger: console,
});

Usa SequelizeStorage que guarda en una tabla SequelizeMeta que migraciones ya se ejecutaron.

Ejemplo de migracion

src/migrations/001-extra-columns-candidatos-perfil.js 
module.exports = {
  up: async ({ queryInterface }) => {
    const tableInfo = await queryInterface.describeTable('candidatos_perfil');
    const columnsToAdd = [
      { name: 'observacion_perfil', type: DataTypes.TEXT },
      { name: 'fecha_observacion',  type: DataTypes.DATE },
      { name: 'estado_laboral',
        type: DataTypes.ENUM('No especifica', 'En busqueda laboral', 'Contratado', 'Freelance'),
        defaultValue: 'No especifica' },
    ];
    for (const col of columnsToAdd) {
      if (!tableInfo[col.name]) {
        await queryInterface.addColumn('candidatos_perfil', col.name, { type: col.type });
      }
    }
  },
  down: async ({ queryInterface }) => {
    const cols = ['estado_laboral', 'acepta_autorizacion', 'fecha_observacion', 'observacion_perfil'];
    for (const col of cols) {
      try { await queryInterface.removeColumn('candidatos_perfil', col); }
      catch { }
    }
  },
};

Sync vs Migrations — Cuando usar cada uno

sequelize.sync()Umzug (migraciones)
Que creaTablas enteras desde modelosColumnas, indices, datos, cambios especificos
IdempotenteSi — SKIP IF NOT EXISTSSi — cada migracion corre una sola vez
VersionadoNo — refleja el modelo actualSi — cada migracion es un commit reversible
RollbackNo aplicaSi — umzug.down()
Seed dataNo recomendadoSi — migraciones de tipo seed
Ideal paraDesarrollo, prototipado, tablas nuevasProduccion, cambios en esquemas existentes
