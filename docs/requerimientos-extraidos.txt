Propuesta de Plataforma Integral: Programa Ágora Argentina Descripción General del Proyecto El Programa Ágora Argentina tiene como misión fundamental la inclusión laboral y la formación profesional de personas con discapacidad visual. Actualmente, la gestión de la información se realiza a través de herramientas externas que limitan la agilidad y la autonomía de los procesos. Este proyecto propone la creación de una plataforma digital propia, centralizada y diseñada desde sus cimientos para ser totalmente accesible. El objetivo es transformar la manera en que los candidatos se inscriben, gestionan su información y acceden a oportunidades laborales y de capacitación, brindando al equipo de Ágora una herramienta potente para conectar el talento con las empresas. Con esta plataforma, se busca eliminar barreras tecnológicas, asegurar la privacidad de los datos sensibles y ofrecer un entorno profesional donde cada usuario pueda autogestionar su camino hacia el empleo. El Camino del Desarrollo: Las 4 Etapas Para asegurar la calidad y el éxito de la implementación, el proyecto se divide en cuatro fases progresivas: 1. Gestión de Perfiles y Documentación (Alcance Actual): Creación del corazón del sistema, donde se registra y valida a los usuarios. 2. Gestión de Cursos y Formación: Un espacio para organizar capacitaciones, inscripciones y reportes de progreso. 3. Gestión de Entrevistas: Herramientas para coordinar encuentros, calendarios y seguimientos personalizados. 4. Perfil Profesional y Conexión Laboral: Un constructor de currículums profesional y un sistema de filtros para que los reclutadores encuentren los perfiles adecuados. Detalle de la Primera Etapa: El Registro y la Validación Esta primera fase es la más importante, ya que establece la base de datos de todas las personas que forman parte del programa. El alcance de esta etapa se centra en los siguientes puntos: Un Portal para el Usuario Se creará un espacio personal donde cada persona podrá registrarse de manera sencilla. En lugar de completar un formulario largo de una sola vez, el sistema permitirá cargar la información por partes, guardando el progreso para que el usuario pueda volver cuando prefiera. Gestión Segura de Documentos Los usuarios podrán subir de forma digital sus documentos esenciales, como el DNI, el Certificado Único de Discapacidad (CUD) y su Currículum Vitae. El sistema contará con un visor que permitirá a los gestores de Ágora revisar estos documentos rápidamente sin necesidad de descargarlos constantemente, garantizando que la información sensible esté siempre protegida. Niveles de Validación Para que la gestión sea más eficiente, los perfiles pasarán por dos estados: Pre-aprobado: Indica que la persona ya inició su carga y puede empezar a participar en algunas actividades, aunque falte algún dato menor. Aprobado: Significa que el perfil está completo, verificado por el equipo de Ágora y listo para ser presentado ante oportunidades de empleo o cursos avanzados. Panel de Gestión para el Equipo de Ágora Los coordinadores y gestores del programa tendrán una herramienta para buscar personas según sus habilidades, ubicación o nivel de estudio. Esto permitirá responder con rapidez cuando una empresa busque un perfil específico o cuando se necesite convocar a un grupo para una capacitación puntual. Comunicación y Transparencia El sistema registrará cada cambio y nota importante. Si un gestor necesita que un usuario actualice su información, podrá enviarle una notificación directa dentro de la plataforma. De esta manera, el usuario siempre sabrá en qué estado se encuentra su perfil y qué pasos debe seguir. Resumen de beneficios inmediatos Autonomía: El usuario es dueño de su información y puede actualizarla cuando lo necesite. Accesibilidad: Todo el sitio está pensado para que sea fácil de usar con lectores de pantalla y otras ayudas técnicas. Organización: El equipo de Ágora deja de usar múltiples archivos sueltos para tener todo en un solo lugar seguro. Seguridad: La información confidencial solo es vista por las personas autorizadas, quedando registro de quién accede a cada dato. Detalle funcional: Esta es la descripción detallada del funcionamiento de la plataforma, enfocada exclusivamente en lo que permite hacer y cómo interactúan las personas con ella. 1. Acceso y Gestión de Identidad El sistema es la puerta de entrada para todos los miembros del programa. Identificación Flexible: Se puede ingresar usando el número de DNI o el correo electrónico, junto con una contraseña personal. Doble Función (Multi-rol): Algunas personas, como los gestores, también pueden ser candidatos. El sistema les permite alternar su vista: pueden estar trabajando en la administración y, con un solo clic, pasar a ver su propio perfil como lo vería cualquier otro candidato. 2. El Portal del Candidato (Su Espacio Personal) Es el área donde la persona con discapacidad visual gestiona su relación con Ágora. Carga por Etapas: No es necesario completar todo el registro de una vez. La información se divide en secciones (Datos Personales, Salud y Autonomía, Educación y Tecnología, Situación Laboral). Un indicador de porcentaje muestra cuánto falta para terminar. Niveles de Validación: El perfil pasa por dos estados claros: Pre-aprobado: El usuario ya puede figurar en el sistema aunque le falte algún detalle. Aprobado: El equipo de Ágora ha verificado que todo está correcto y completo, habilitando al usuario para todas las funciones del programa. Autogestión de Documentos: El usuario sube sus propios archivos (DNI, CUD, CV). Si necesita actualizarlos, los elimina y sube los nuevos. 3. El Centro de Gestión (SSC - Panel para Administradores) Es el cerebro operativo donde los gestores y coordinadores transforman los datos en oportunidades. Buscador Inteligente: Permite encontrar personas rápidamente. Se pueden aplicar filtros por ciudad, por tipo de discapacidad o por conocimientos específicos. Filtros Flexibles: Se puede buscar personas que cumplan &quot;al menos uno&quot; de los requisitos seleccionados o, de forma más estricta, personas que cumplan &quot;con todos&quot; los requisitos al mismo tiempo. Exportación Controlada: Los resultados de una búsqueda se pueden descargar en un archivo Excel. Este archivo incluye los datos de las personas y enlaces directos para ver sus documentos, siempre que el gestor tenga el permiso necesario. 4. Gestión de Documentos y Seguridad El manejo de los archivos es riguroso para proteger la privacidad. Visor Integrado: Los gestores pueden ver una vista previa de los documentos (imágenes o texto) directamente en el sitio, agilizando la revisión sin tener que descargar cada archivo a su computadora. Papelera de Seguridad: Cuando un usuario borra un documento, este no desaparece inmediatamente del sistema. Se guarda en un área oculta por 30 días como medida de respaldo antes de borrarse definitivamente. Límites de Carga: Existe un control sobre el tamaño de los archivos y la cantidad que cada usuario puede subir, para mantener el sistema organizado y eficiente. Libro de Visitas (Auditoría): El sistema anota automáticamente quién vio cada documento, qué documento fue y en qué momento exacto ocurrió. 5. Notas y Observaciones Internas Es el espacio de trabajo privado para los orientadores del programa. Expediente del Candidato: Los gestores pueden dejar notas sobre el progreso de una persona o detalles de su perfil. Estas notas pueden incluir enlaces o archivos adicionales. Privacidad Absoluta: Estas notas son totalmente invisibles para el candidato; solo las pueden leer y gestionar los administradores y coordinadores. 6. Personalización y Accesibilidad El sitio se adapta a las necesidades visuales de cada usuario. Ajuste Visual: Cada persona puede elegir si prefiere ver el sitio en modo oscuro o modo claro, y puede aumentar el tamaño de la letra para una lectura más cómoda. Memoria de Preferencias: El sistema recuerda estas opciones automáticamente. Si el usuario cambia el tamaño de letra en su computadora, cuando entre desde su teléfono el sitio se verá igual. 7. Respaldo de la Información Para asegurar que nunca se pierda un dato importante: Copias de Seguridad: El sistema realiza respaldos automáticos de toda la información de los perfiles cada pocos días. Almacenamiento Redundante: Los documentos se guardan en el servidor principal y también se envía una copia a un servicio de respaldo en la nube para mayor seguridad (opcional y configurable, a definir). Detalle técnico: Esta es la propuesta técnica detallada para el desarrollo de la plataforma, diseñada bajo estándares de escalabilidad, seguridad y accesibilidad universal. Propuesta Técnica: Plataforma de Gestión Ágora Argentina 1. Ecosistema de Desarrollo y Tecnologías El sistema se desarrollará como una aplicación web robusta utilizando una arquitectura de software probada que garantiza estabilidad y facilidad de mantenimiento. Lenguaje y Framework Principal: Python con Django. Este entorno permite una gestión segura de datos y un desarrollo modular mediante aplicaciones independientes. Base de Datos: MySQL. Se utilizará para el almacenamiento relacional de perfiles, documentos y registros de auditoría, asegurando la integridad de la información. Interfaz de Usuario: Bootstrap para el diseño estructural y Alpine.js para la interactividad reactiva (como el sistema de notificaciones y la actualización de perfiles sin recargar la página). Estilos y Personalización: Se utilizará un núcleo de CSS centralizado mediante variables (Custom Properties). Esto permitirá el control global del diseño, facilitando la implementación del modo oscuro y el escalado de fuentes desde un solo punto de configuración. 2. Estándares de Accesibilidad (WCAG 2.2) El desarrollo se regirá estrictamente por las pautas de accesibilidad para el contenido web en su versión 2.2, nivel AA/AAA. Navegación: Estructura de encabezados lógica, saltos de navegación y foco de teclado claramente visible. Adaptabilidad: Diseño responsivo que mantiene la funcionalidad en cualquier tamaño de pantalla. Contraste y Visualización: Soporte nativo para modo oscuro y configuración de tamaño de fuente persistente (mediante cookies y sesión de usuario) para usuarios con baja visión. Semántica: Uso de etiquetas HTML5 semánticas y atributos ARIA para asegurar que los lectores de pantalla (NVDA, JAWS) interpreten correctamente cada sección. 3. Infraestructura y Requisitos del Servidor Para el despliegue y funcionamiento óptimo del servicio, se requiere un entorno de servidor con las siguientes características: Sistema Operativo: Servidor basado en Linux (preferentemente Ubuntu Server o Debian). Requisitos de Hardware Sugeridos: Procesador: 2 vCPU o superior. Memoria RAM: 4 GB mínimo para manejar procesos de encriptación y gestión de archivos. Almacenamiento: Espacio SSD escalable según la carga de documentos (DNI, CUD, CV). Servidor Web y de Aplicaciones: Nginx como servidor proxy inverso y Gunicorn para ejecutar la lógica de Django. Seguridad y Acceso: Se requiere acceso al servidor mediante SSH para la configuración inicial. El despliegue se realizará a través de Git, permitiendo actualizaciones controladas y reversión de versiones en caso de errores. 4. Ciclo de Vida y Metodología de Trabajo El proyecto no se entrega como un bloque cerrado al final, sino como un proceso de construcción transparente. Metodología: Scrum. El trabajo se dividirá en etapas de desarrollo (Sprints) con una duración de 3 semanas cada una. Al finalizar cada etapa, se entregará una parte funcional del sistema para su revisión y prueba. Entrega Final: Se entregará el sitio completamente operativo en el servidor de producción, con todas las funcionalidades de la Fase 1 validadas, la base de datos normalizada y los scripts de respaldo configurados. 5. Documentación y Estándares de Código Siguiendo las mejores prácticas, cada componente del sistema estará debidamente documentado para su mantenimiento futuro. Documentación Funcional (El Contrato): Cada función o clase técnica detallará su propósito, los argumentos que recibe (tipo y significado) y qué valor devuelve. Comentarios: Se incluirán aclaraciones directas sobre variables y partes críticas de los algoritmos de forma simplista para facilitar la lectura con tecnologías de asistencia. Sugerencias de Buenas Prácticas y Optimización 2. Responsabilidad Única: Los módulos de gestión de archivos y encriptación deberán estar aislados de la lógica de usuario para facilitar auditorías de seguridad independientes. 3. Seguridad de Archivos: Implementar una validación de tipo MIME real para las subidas, evitando que archivos maliciosos sean renombrados y subidos como PDFs o imágenes. 4. Escalabilidad de CSS: Al centralizar el diseño en un solo archivo CSS con variables, se documentará cada variable (ej: `--main-bg-color`) para que futuros ajustes estéticos no rompan la accesibilidad del sitio. Stack de Librerías y Dependencias de Django Para garantizar que el sistema sea modular, seguro y escalable, se utilizarán las siguientes librerías de Python/Django: django-storages: Fundamental para la gestión de archivos híbrida. Permite conectar Django con servicios como Google Cloud Storage o Amazon S3 sin cambiar la lógica del código. google-api-python-client &amp; google-auth: Para la integración directa con la API de Google Drive, permitiendo crear carpetas de respaldo y subir archivos de forma automatizada. cryptography / django-cryptography: Utilizadas para el cifrado de datos sensibles (como el DNI o el CUD) antes de ser guardados en la base de datos MySQL, asegurando que solo el sistema pueda leerlos. celery &amp; redis: Para la gestión de tareas asíncronas. Esto es crítico para realizar los backups sin detener el sitio y para procesar las llamadas a modelos de IA (Gemini/OpenAI) que pueden tardar unos segundos. django-dbbackup: Herramienta para automatizar los respaldos de la base de datos MySQL y de los archivos multimedia, permitiendo programar copias hacia el servidor local o la nube. django-environ: Gestión de variables de entorno. Permite separar las claves API (Gemini, OpenAI, Drive) del código fuente, aumentando la seguridad. google-generativeai: La librería oficial para interactuar con Gemini. Se usará para el análisis de documentos y asistencia en la accesibilidad. openai: Implementada como respaldo para funciones de procesamiento de lenguaje natural y análisis de perfiles profesionales. Integración con Servicios Cloud e IA La plataforma operará bajo una lógica de &quot;Nube Híbrida&quot;: Google Drive API: Se configurará un &quot;Service Account&quot; para que el sistema tenga su propio espacio de almacenamiento. Cada vez que un usuario suba un documento, el sistema lo guardará localmente (con cifrado) y enviará una copia a una carpeta específica en Drive como backup redundante. Modelos de IA (Gemini / OpenAI): Función: Procesamiento de documentos (OCR opcional) y generación de resúmenes de perfil. Implementación: Se utilizarán &quot;Hooks&quot; de Django. Al subir un archivo, Celery disparará una tarea en segundo plano que enviará el documento a la API de Gemini para extraer texto o validar que el archivo sea legible, devolviendo el resultado al panel del gestor. Requisitos y Configuración del Servidor El despliegue requiere un entorno controlado para asegurar que las políticas de accesibilidad y seguridad se mantengan: Acceso SSH: Indispensable para la configuración de seguridad del servidor (firewall), instalación de certificados SSL y gestión de procesos en tiempo real. Control de Versiones (Git): El código se gestionará mediante Git. El flujo de trabajo implica un entorno de &quot;Staging&quot; (pruebas) y uno de &quot;Producción&quot;. Cada actualización de funcionalidad se subirá primero para validación de accesibilidad antes de pasar al sitio en vivo. Despliegue (Gunicorn + Nginx): Nginx actuará como la cara visible (Proxy), gestionando el cifrado HTTPS y los archivos estáticos (CSS/JS), mientras que Gunicorn ejecutará la lógica de Django de forma eficiente. Se aplicará la metodología Scrum con entregas parciales (Sprints) cada 3 semanas. 2. Cronograma de Trabajo (12 Semanas) Sprint 1: Cimientos y Seguridad (Semanas 1 a 4) Backend: Configuración de servidor (SSH/Nginx/Git), base de datos MySQL, Modelo de Usuario Personalizado (DNI/Email), lógica de roles y permisos jerárquicos. Frontend: Estructura base de diseño, configuración de variables CSS para accesibilidad, sistema de navegación accesible y pantallas de Login/Registro. Entrega: Sistema de autenticación funcionando y base visual del proyecto instalada en el servidor. Sprint 2: El Corazón del Perfil (Semanas 5 a 8) Backend: Lógica de perfiles (4 etapas de carga), sistema de encriptación de datos sensibles, integración con la API de Google Drive para backups de archivos y lógica de &quot;borrado lógico&quot; de 30 días. Frontend: Formulario progresivo (paso a paso), gestión visual de documentos (subida con progreso), implementación de Dark Mode y selector de tamaño de fuente. Entrega: Candidatos pueden completar su perfil al 100% y subir documentos de forma segura. Sprint 3: Gestión y Herramientas Administrativas (Semanas 9 a 12) Backend: Buscador avanzado (Lógica AND/OR), generación de reportes Excel con links de descarga, automatización de copias de seguridad de la BD, preparación de conexión opcional con IA (Gemini/OpenAI). Frontend: Panel administrativo (SSC), visor de documentos integrado (previsualización), sistema de notificaciones en tiempo real (Polling cada 10 min) y auditoría visual de accesibilidad WCAG 2.2. Entrega: Sistema completo funcional, migración de datos históricos de Google Forms y puesta en marcha final. Al finalizar cada sprint, se entregará una versión funcional para pruebas de usuario. La fase final consistirá en la puesta en marcha definitiva con los datos históricos ya migrados y normalizados. Esta estimación de 12 semanas contempla el desarrollo funcional del sistema. Sin embargo, se prevé una extensión opcional de hasta 4 semanas adicionales destinadas exclusivamente a procesos de testing integral, validaciones de seguridad, pruebas de accesibilidad con usuarios finales y auditorías de usabilidad. Es importante aclarar que este cronograma no incluye tiempos derivados de problemas técnicos ajenos al desarrollo, tales como fallas en la infraestructura del servidor o demoras en su configuración inicial, los cuales se consideran factores externos a la etapa de programación y despliegue. 
Sistema Agora: Este es el detalle funcional y de navegación de la plataforma, diseñado para ser intuitivo y totalmente accesible. El sistema se adapta dinámicamente según quién ingrese y qué permisos tenga. 
Acceso Inicial: Pantalla de Identificación La experiencia comienza en una interfaz despejada y de alto contraste. Campos de ingreso: El usuario puede identificarse ingresando su número de DNI o su correo electrónico, seguido de su contraseña. 		
Recuperación: en caso de olvido, solo el gestor podrá cambiar la contraseña. 
Accesibilidad desde el inicio: Antes de ingresar, el usuario ya puede ajustar el tamaño de la fuente o activar el modo oscuro, y estas preferencias se mantendrán durante toda su sesión. 
El Portal del Candidato: Navegación y Funciones Al ingresar como usuario que busca empleo o formación, la navegación se organiza de la siguiente manera: Panel de Control (Dashboard) Es la página de inicio. Lo primero que verá el usuario es: Estado del Perfil: Una barra de progreso que indica qué porcentaje de su información ha cargado. Notificaciones: Alertas sobre entrevistas agendadas, mensajes de gestores o avisos de que debe actualizar algún documento. Estas notificaciones aparecen automáticamente sin necesidad de recargar la página. Mi Perfil (Carga Progresiva) La navegación se divide en secciones claras para no abrumar al usuario: Datos Personales y Ubicación: Nombre, contacto y residencia. Discapacidad y Autonomía: Detalles sobre su condición visual y nivel de independencia. Formación y Tecnología: Estudios realizados y herramientas que utiliza (como lectores de pantalla). Situación Laboral: Si busca empleo, si tiene un emprendimiento y sus intereses actuales. 
Gestor de Documentos Aquí el usuario puede subir sus archivos (CUD, DNI, CV). Acciones: Subir nuevo documento, ver el documento actual o eliminarlo. Seguridad: Si el usuario elimina un archivo por error, el sistema le informará que el registro queda guardado de forma privada por 30 días antes de su borrado definitivo. 
Panel de Gestión (SSC): Roles y Capacidades Para los usuarios con roles de gestión, la navegación incluye un menú lateral con herramientas avanzadas. Rol de Gestor y Coordinador Buscador Avanzado: Es la herramienta principal. El gestor puede filtrar la base de datos por múltiples criterios (ejemplo: personas que vivan en Salta, sean usuarios de NVDA y tengan el secundario completo). 
Ficha del Candidato: Al entrar en un perfil, el gestor puede: Ver los datos y documentos (usando el visor interno para no descargar archivos). Cambiar el estado de validación (de &quot;Pre-aprobado&quot; a &quot;Aprobado&quot;). Agregar notas privadas con archivos adjuntos. Enviar avisos al usuario solicitando actualizaciones. Diferencia de jerarquía: El Coordinador es el único que puede asignar nuevos roles (convertir a un usuario en Gestor o Instructor). Un Gestor puede operar la base de datos pero no puede elevar el rango de otros usuarios. Rol de Gestor Técnico Este usuario tiene acceso a una sección especial de &quot;Salud del Sistema&quot;: 
Historial Técnico: Ve alertas sobre errores de carga de archivos o fallos en las comunicaciones. Configuración Global: Es quien define el tamaño máximo de los archivos permitidos (ejemplo: 5MB) y gestiona los catálogos de países, provincias y habilidades técnicas. Navegación Interactiva y Global 
Cambio de Vista (Multi-rol) Si un Gestor también es Candidato, verá en la parte superior un botón de &quot;Cambiar a modo Candidato&quot;. Al pulsarlo, el menú de administración desaparece y el sitio se transforma completamente en el portal de usuario, permitiéndole gestionar su propio perfil con privacidad. 
Personalización de la Interfaz En todo momento, en la barra de navegación superior, estarán disponibles los controles de: Modo Oscuro/Claro: Para descanso visual o mejor contraste. Tamaño de Letra: Permite agrandar el texto en tres niveles distintos. 
Cerrar Sesión: Salida segura del sistema. 
Notificaciones In-App Cada 10 minutos, el sistema verifica si hay novedades. Si aparece una nueva notificación, un indicador sonoro y visual avisará al usuario. El usuario puede marcar las notificaciones como leídas o pulsar botones dentro de la alerta para ir directamente a la sección que requiere su atención. 
ADMINISTRACIÓN DE DATOS MAESTROS (EL ALMACÉN DE INFORMACIÓN) Esta es una de las partes más potentes del panel. El gestor tiene el control total sobre las opciones que el usuario ve al registrarse. Gestión de Catálogos (CRUD): Desde el panel se pueden dar de alta, modificar o dar de baja: - Países, provincias y ciudades. - Tipos de discapacidad y condiciones. - Habilidades técnicas (como lectores de pantalla NVDA, JAWS, etc.). - Niveles educativos y carreras. Regla de Integridad (Habilitar y Deshabilitar): Para proteger la base de datos, si una opción ya está siendo usada por un usuario (por ejemplo, &quot;Provincia de Buenos Aires&quot;), el gestor no podrá eliminarla. En su lugar, podrá &quot;Deshabilitarla&quot;. - Si se deshabilita: Ya no aparecerá como opción para nuevos registros. 
- Notificaciones de Actualización: El sistema permite ver cuántos usuarios tienen asignada una opción que ahora está deshabilitada. Desde ahí, el gestor puede enviar un aviso masivo a esas personas para que entren a su perfil y elijan una opción nueva o actualizada. -
----EL BUSCADOR AVANZADO Y EXPORTACIÓN A EXCEL Esta herramienta transforma los datos en gestión real de empleo. Filtros de Precisión: El buscador permite combinar criterios geográficos, educativos y de habilidades. Se puede activar la opción de que el resultado deba cumplir obligatoriamente con todos los filtros marcados (Lógica Y) o que cumpla con al menos uno de ellos (Lógica O). Descarga de Reportes en Excel: Una vez aplicados los filtros, el gestor puede bajar esa lista específica en un archivo de Excel. - Contenido del Excel: El archivo tendrá todos los datos del perfil filtrado. 
- Enlaces de Acceso: El Excel incluirá enlaces directos para ver o descargar los documentos (DNI, CUD, CV). Estos enlaces solo funcionarán si quien los abre tiene una sesión iniciada y permisos suficientes en la web. -----GESTIÓN DE USUARIOS Y VALIDACIÓN Control de Cuentas: El gestor puede buscar a cualquier usuario por su nombre o DNI para: - Suspender cuentas (por mal uso o inactividad). 
- Reactivar cuentas. - Cambiar el nivel de validación (de Pre-aprobado a Aprobado). Historial de Notas y Expediente: Al entrar al perfil de un usuario, el gestor tiene una sección de &quot;Notas de Seguimiento&quot;. - Estas notas permiten usar un formato de texto enriquecido para agregar enlaces o imágenes. - Se pueden adjuntar archivos adicionales a estas notas que solo los gestores podrán ver. - Cada nota queda guardada con fecha y autor, creando un historial completo de la atención brindada a esa persona. CONTROL TÉCNICO Y CONFIGURACIÓN DE ARCHIVOS El Gestor Técnico tiene herramientas adicionales para asegurar la salud del sitio: Límites de Espacio: Puede configurar el tamaño máximo de los archivos que se permiten subir (por ejemplo, 5 MB por documento) y la cantidad máxima de archivos por persona. Alertas Técnicas: Si hay un error en el servidor o un problema al intentar guardar un archivo en la nube (Google Drive), el gestor técnico recibirá una alerta en su panel para corregirlo de inmediato. Registro de Auditoría: Se puede consultar el &quot;Log de Actividades&quot;, donde queda registrado qué gestor entró a qué perfil y si descargó o visualizó documentos sensibles. 
ESTADOS LABORALES Y OBSERVACIONES El sistema permite gestionar el estado de los candidatos para la intermediación: - El usuario puede marcarse como: En búsqueda laboral, Contratado o Freelance. - Si el gestor nota que a un perfil le falta información crítica para ser presentado a una empresa, puede marcar una &quot;Observación de Perfil Incompleto&quot;. Esta observación queda como una alerta naranja en el panel del gestor para recordarle que debe dar seguimiento a ese caso. Este nivel de detalle asegura que el equipo de Ágora no solo tenga una lista de nombres, sino una plataforma de gestión profesional donde cada dato puede ser controlado, actualizado y exportado con seguridad. 
Arquitectura de la Base de Datos (Modelo Relacional) La base de datos en MySQL se organizará de forma modular para reflejar la estructura de aplicaciones de Django. No solo guardará texto, sino que gestionará estados y permisos complejos. Núcleo de Usuarios y Roles Usuarios: Almacena la identidad mínima necesaria (DNI o Email y contraseña cifrada). Roles: Una tabla puente que permite que un usuario sea &quot;Candidato&quot; y &quot;Gestor&quot; simultáneamente. Sesiones y Preferencias: Tabla para recordar si el usuario prefiere el modo oscuro o un tamaño de fuente específico, vinculando estos datos a su cuenta. El Perfil Maestro (Candidato) El perfil no es una tabla única gigante, sino un conjunto de tablas relacionadas para mantener el orden: Datos de Perfil: Relacionada 1 a 1 con el usuario. Contiene el estado de validación (Pre-aprobado/Aprobado) y el porcentaje de completitud. Secciones de Información: Tablas separadas para Educación, Experiencia Laboral y Datos de Discapacidad. Esto permite que el usuario guarde &quot;borradores&quot; de cada sección. Tabla de Notas: Notas privadas escritas por gestores, con soporte para formato Markdown y archivos adjuntos internos. Catálogos y Datos Maestros (Normalización) Para evitar errores de escritura, todo lo que sea una opción de selección (países, habilidades como NVDA, niveles de estudio) tendrá su propia tabla. Atributo de Estado: Cada opción tendrá un campo &quot;Activo/Inactivo&quot;. Integridad: Se programará una restricción para que no se pueda eliminar una opción si ya hay perfiles usándola; solo se podrá deshabilitar para futuros registros. Gestión de Archivos y Auditoría Metadatos de Archivos: Guardará la ruta del archivo, el tipo (CUD, DNI, CV), el proveedor (Local o Google Drive) y una marca de tiempo para el &quot;borrado lógico&quot; de 30 días. Logs de Auditoría: Una tabla dedicada a registrar cada &quot;Evento&quot;. Ejemplo: `[Fecha] - [Usuario Gestor X] - [Acción: Visualizó CUD] - [Candidato Y]`. Consideraciones Técnicas para la Programación Para que el equipo de desarrollo (Backend y Frontend) trabaje alineado, debemos respetar estos pilares técnicos: Seguridad y Privacidad de Datos Cifrado a Nivel de Campo: Los datos sensibles (como números de documento) deben cifrarse antes de entrar a MySQL. Si alguien accede a la base de datos directamente, no podrá leer esta información sin la llave maestra del servidor. Protección de Archivos: Los documentos nunca deben estar en una carpeta de acceso público. Se debe programar una &quot;Vista Intermediaria&quot; en Django que verifique los permisos antes de entregar el archivo al navegador. Desarrollo Modular y Accesibilidad Apps Independientes: El código de Django debe estar dividido (ej. app `usuarios`, app `perfiles`, app `notificaciones`). Esto facilita que en la Fase 2 se pueda agregar el módulo de cursos sin romper lo anterior. CSS Centralizado: El Frontend debe usar Variables CSS, para que el cambio a &quot;Modo Oscuro&quot; o el aumento de fuente se propague por todo el sistema de forma instantánea y coherente. Lógica de Sincronización (Cloud &amp; Backups) Tareas en Segundo Plano: El proceso de subir el backup a Google Drive o realizar el respaldo de la base de datos no debe bloquear la web. Se programará para que ocurra de forma asíncrona (en segundo plano). Sistema de Notificaciones (Polling): La interfaz de Alpine.js consultará al servidor cada 10 minutos por nuevas alertas. Esta consulta debe ser muy ligera (solo traerá el número de alertas nuevas) para no saturar el servidor.
